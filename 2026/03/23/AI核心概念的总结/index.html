AI核心概念的总结

发表于2026-03-23|更新于2026-03-23|文章
|浏览量:

大模型基础概念

LLM (Large Language Model)，中文名称大语言模型，即大模型。

  • 大模型是基于 Transformer 架构训练出来的。
  • 大模型最底层的生成原理是**提示词链 (Prompt Chaining)**。

Tokenizer —— 负责编码和解码，即人类和大模型之间的“翻译工具”。

编码是把文字转换成数字，解码是把数字转换成文字。

  • 编码的步骤：分为切分和映射。切分是指将一句话分成多个片段，也称为 Token；由于大模型只识别数字，Tokenizer 就会把每一个 Token 对应到一个 TokenID 上去。
  • 解码的步骤：只有一步——映射，即把 TokenID 转换为文字。

注意： 一个 Token 并不一定对应一个词。Token 可以理解为模型自己学会的一套文本切分规则，切分的每一个部分就是它一次能够处理的最小单位。

上下文与记忆

  • **Context (上下文)**：即大模型每次处理任务时所接收到的信息总和（比如用户与 AI 的对话历史和用户问题都是 Context 的一部分）。从某种程度上来说，我们也可以把它看作是大模型的一个临时记忆体。
  • **Context Window (上下文窗口)**：即 Context 能容纳的最大 Token 数量。

RAG (检索增强生成)

**RAG (Retrieval-Augmented Generation)**，即检索增强生成。

这个概念可以概述为：先从资料库里检索相关内容，再基于这些内容来生成答案。

  • 使用场景：可以形象地概述为用来解决“产品手册太长所带来的问题”的解决方案。

RAG 的基本运行流程：

  1. **分片 (Chunking)**：即把一个长文档切分成多个部分。
  2. 索引 (Indexing)：通过 Embedding 将片段文本转换为向量，再将片段向量存入向量数据库的过程。
    • 向量可以用数组的形式来表示。Embedding 即为把文本转化为向量的过程。
    • 比如用户提问了一个问题，模型就会将这个问题做一个 Embedding，将其转换为向量，然后根据向量相似度，把与这个问题相关的文本也找出来。Embedding 后的向量会储存在向量数据库中，方便用户查询。

  3. **重排 (Reranking)**：使用一种叫做 cross-encoder 的模型计算每个片段与用户问题的相似度并重新排序。
  4. **生成 (Generation)**：大模型根据用户问题和检索出来的相关片段，来回答用户的问题。

提示词与工具调用

**Prompt (提示词)**，是大模型接收的具体问题或指令。主要分为两类：

  • **User Prompt (用户提示词)**：用户自己输入的，用来说明具体任务。
  • **System Prompt (系统提示词)**：开发者在后台配置的，用来说明人设和做事规则。

扩展能力：Tool 与 Agent

  • **Tool (工具)**：功能就是让大模型能够感知外部环境、获取实时信息或执行动作。
  • **MCP (Model Context Protocol)**：即模型上下文协议。它是一套统一的工具接入标准，工具的开发者只需要按照 MCP 的规范开发一套工具，就可以被所有支持 MCP 的平台使用了。
  • Agent (智能体)：简单来说，一种能够自主规划、自主调用工具直至完成用户任务的系统就是 Agent（如 Claude Code、Codex、Gemini CLI 等）。Agent 是由构建模式来设计的。
  • Agent Skill：简单来说就是一个给 Agent 看的说明文档。
